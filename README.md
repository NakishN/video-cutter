# 🎬 Нарезчик видео

Автоматическая транскрипция Twitch-VOD и YouTube-видео с выделением ключевых моментов с помощью ИИ.

**Как работает:**
1. **Загрузка:** Загружаете видео/аудио файл с ПК или вставляете ссылку на Twitch / YouTube (приложение скачает только аудиодорожку, что экономит трафик и ускоряет процесс **в 10–50 раз**).
2. **Транскрипция:** [Whisper](https://github.com/ggerganov/whisper.cpp) транскрибирует речь локально (на видеокарте GPU с CUDA или на процессоре CPU с автоматическим резервированием).
3. **Анализ:** [GenAPI](https://gen-api.ru) (GPT-4.1 или Gemini) находит самые интересные, эмоциональные моменты и выдаёт список с таймкодами и описанием.
4. **Экспорт:** Скачиваете готовый архив с текстом реплик, таймкодами (SRT) и резюме.

---

## ✨ Новые возможности и улучшения

* **Премиальный интерфейс:** Полностью обновлённый дизайн в тёмных тонах с эффектом Glassmorphism (матовое стекло) и адаптивной версткой.
* **Поддержка YouTube:** Ввод ссылок теперь полноценно работает как для Twitch, так и для YouTube с автоматическим распознаванием сервиса.
* **Drag & Drop:** Удобная область для перетаскивания файлов с ПК прямо в браузер с мгновенным отображением размера файла.
* **Индикаторы статуса:** Информативные значки в шапке показывают текущее состояние GPU-ускорения (CUDA) и подключения к облаку GenAPI.
* **Вкладки результатов с быстрым копированием:** Полученный текст реплик, файлы субтитров (.srt) и резюме LLM разделены по вкладкам, и их можно скопировать в буфер обмена в один клик.
* **Очистка кэша:** Специальная кнопка в боковой панели позволяет мгновенно удалить временные аудио- и видеофайлы с диска, высвобождая дисковое пространство.
* **Отказоустойчивость:** Автоматический откат на CPU при сбое CUDA/GPU на этапе транскрипции.

## 🪟 Windows — Подробная инструкция (для новичков)

Эта инструкция написана специально для тех, кто редко сталкивается с установкой программ из исходных кодов. Мы разберём как автоматический, так и полностью ручной способ.

### 🚀 Способ 1: Автоматическая установка (через `Запустить.bat`)

Самый простой способ, но иногда он может давать сбои (особенно если антивирус или настройки Windows блокируют скачивание файлов).

**Шаг 1. Установка Python**
1. Перейдите на официальный сайт Python: [python.org/downloads](https://www.python.org/downloads/)
2. Нажмите большую жёлтую кнопку **Download Python**.
3. Запустите скачанный установщик.
4. **КРИТИЧЕСКИ ВАЖНО:** В самом низу первого окна установки обязательно поставьте галочку **`Add python.exe to PATH`** (Добавить в PATH). Без этого ничего работать не будет!
5. Нажмите **`Install Now`** и дождитесь окончания установки.

**Шаг 2. Скачивание программы**
1. Нажмите зелёную кнопку **Code → Download ZIP** вверху этой страницы.
2. Распакуйте архив в удобное место. **Важно:** путь к папке не должен содержать русских букв (лучше `C:\VideoCutter` вместо `C:\Мои Документы\Нарезчик`).

**Шаг 3. Запуск**
1. Зайдите в распакованную папку и дважды кликните файл `Запустить.bat`.
   > *Если Windows выдаст синее окно "Система Windows защитила ваш ПК", нажмите **`Подробнее`**, а затем **`Выполнить в любом случае`**.*
2. При первом запуске скрипт сам создаст виртуальное окружение, установит зависимости и попросит API-ключ.
3. Зарегистрируйтесь на сайте [gen-api.ru](https://gen-api.ru), скопируйте ключ (начинается на `sk-`) и вставьте его в появившийся файл `.env`. Сохраните файл (`Ctrl+S`) и закройте.
4. Программа сама откроется в браузере (`http://127.0.0.1:8000`).

### 🛠️ Способ 2: Полностью ручной метод (если `.bat` выдаёт ошибки)

Иногда встроенные системы защиты Windows (PowerShell) блокируют скачивание файлов скриптами (`setup_windows.bat`). Если скрипт закрывается с ошибкой, мы всё установим вручную. Это надёжнее!

**Шаг 1. Установка Python**
Сделайте так же, как описано в Шаге 1 Автоматического способа. **Не забудьте галочку `Add python.exe to PATH`!**

**Шаг 2. Скачивание программы**
Скачайте ZIP-архив с кодом (зеленая кнопка Code -> Download ZIP) и распакуйте его, например, в папку `C:\VideoCutter`.

**Шаг 3. Ручное скачивание FFMPEG (для работы с видео)**
1. Перейдите по ссылке: [FFmpeg Builds (GitHub)](https://github.com/BtbN/FFmpeg-Builds/releases/latest)
2. Найдите в списке файл **`ffmpeg-master-latest-win64-gpl.zip`** и скачайте его.
3. Откройте скачанный архив, зайдите в папку `ffmpeg-master-latest-win64-gpl`, затем в папку `bin`.
4. Скопируйте файлы **`ffmpeg.exe`** и **`ffprobe.exe`** и вставьте их **прямо в папку с нашей программой**.

**Шаг 4. Ручное скачивание модели искусственного интеллекта**
1. Зайдите в папку программы и создайте там новую папку с названием **`models`** (если её там ещё нет).
2. Перейдите по ссылке: [Скачать ggml-medium.bin (1.5 ГБ)](https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-medium.bin)
3. Поместите скачанный файл `ggml-medium.bin` в папку **`models`**.

**Шаг 5. Настройка ключа GenAPI**
1. В папке с программой найдите файл **`.env.example`**. Скопируйте его и переименуйте в **`.env`**.
2. Получите ключ на [gen-api.ru](https://gen-api.ru).
3. Откройте файл `.env` через Блокнот и напишите ваш ключ: `GEN_API_KEY=sk-ваш_ключ`.
4. Сохраните и закройте файл.

**Шаг 6. Ручной запуск через командную строку**
1. Откройте папку с программой.
2. В строке пути (наверху) удалите текст, напишите **`cmd`** и нажмите **Enter**. Откроется чёрная консоль.
3. Введите по очереди команды:

   ```cmd
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   python launcher.py
   ```

4. Откройте в браузере `http://127.0.0.1:8000`. В будущем достаточно запускать `venv\Scripts\activate` и `python launcher.py`.

## 🐧 Linux — локальный запуск

### Требования

- Ubuntu 20.04+ / Debian 11+ / любой Linux
- Python 3.9+
- ffmpeg
- CUDA (опционально, для ускорения Whisper)

### Шаг 1 — Системные зависимости

```bash
sudo apt-get update
sudo apt-get install -y ffmpeg zip wget git cmake make g++ libopenblas-dev
```

### Шаг 2 — Скачайте проект

```bash
git clone https://github.com/NakishN/video-cutter.git
cd video-cutter
```

### Шаг 3 — Python-окружение

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Шаг 4 — Скомпилируйте Whisper

```bash
cd whisper.cpp
cmake -B build -DGGML_BLAS=ON -DGGML_BLAS_VENDOR=OpenBLAS
cmake --build build --config Release -j$(nproc)
cp build/bin/whisper-cli ../whisper
cd ..
```

> Если есть NVIDIA GPU:
> ```bash
> cmake -B build -DGGML_CUDA=ON
> cmake --build build --config Release -j$(nproc)
> cp build/bin/whisper-cli ../whisper-cuda
> ```
> И поставьте `"use_gpu": true` в `config.json`.

### Шаг 5 — Скачайте модель Whisper

```bash
mkdir -p models
wget https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-medium.bin \
     -O models/ggml-medium.bin
```

Альтернативные модели (меньше = быстрее, но хуже качество):

| Модель | Размер | Качество | Скорость (CPU) |
|---|---|---|---|
| `ggml-tiny.bin` | 75 МБ | низкое | ~2 мин/час видео |
| `ggml-base.bin` | 142 МБ | среднее | ~5 мин/час видео |
| `ggml-small.bin` | 466 МБ | хорошее | ~10 мин/час видео |
| `ggml-medium.bin` | 1.5 ГБ | **отличное** | ~20 мин/час видео |

### Шаг 6 — Настройте .env

```bash
cp .env.example .env
nano .env
```

Вставьте ключ:
```
GEN_API_KEY=sk-ваш-ключ
```

### Шаг 7 — Запуск

```bash
bash start.sh
```

Откройте в браузере: `http://localhost:8000`

## 🌐 Linux — деплой на VPS (Docker)

Подходит если хотите использовать сервис круглосуточно из браузера с любого устройства.

### Требования к серверу

| | Минимум | Рекомендуется |
|---|---|---|
| CPU | 2 ядра | 4 ядра |
| RAM | 4 ГБ | 8 ГБ |
| Диск | 10 ГБ | 20 ГБ |
| ОС | Ubuntu 22.04 | Ubuntu 22.04 |

**Стоимость:** ~1 000–1 500 ₽/мес (Selectel, Timeweb, Hetzner).

### Деплой одной командой

Скопируйте проект на сервер и выполните:

```bash
# С вашей локальной машины — копируем проект на сервер
scp -r /путь/к/video-cutter user@IP_СЕРВЕРА:/opt/video-cutter

# Подключаемся к серверу
ssh user@IP_СЕРВЕРА

# Запускаем деплой
cd /opt/video-cutter
GEN_API_KEY=sk-ваш-ключ sudo -E bash deploy.sh
```

Скрипт сам:
- Установит Docker
- Скачает модель Whisper (~1.5 ГБ)
- Соберёт Docker-образ (5–10 мин при первом запуске)
- Запустит контейнер
- Проверит работоспособность

После запуска приложение доступно по адресу: `http://IP_СЕРВЕРА:8000`

### Управление контейнером

```bash
# Посмотреть статус
docker compose ps

# Посмотреть логи
docker compose logs -f

# Перезапустить
docker compose restart

# Остановить
docker compose down

# Обновить после изменений в коде
docker compose up -d --build
```

### Возможные проблемы (Linux / Docker)

| Проблема | Решение |
|---|---|
| `GEN_API_KEY не задан` | Отредактируйте `/opt/video-cutter/.env`, добавьте ключ, перезапустите |
| Нет места на диске | Воспользуйтесь кнопкой «Очистить кэш видео» в веб-интерфейсе или выполните: `rm /opt/video-cutter/videos/*` |
| Контейнер падает | Проверьте логи: `docker compose logs --tail=50` |
| Порт 8000 занят | Измените в `docker-compose.yml`: `"8080:8000"` |

## ⚙️ Настройки (config.json)

```json
{
  "whisper_model_path": "models/ggml-medium.bin",
  "genapi_network_id": "gpt-4-1",
  "models_dir": "models",
  "use_gpu": false,
  "whisper_language": "ru",
  "video_dir": "videos",
  "output_dir": "output",
  "temp_dir": "tmp"
}
```

| Параметр | Описание |
|---|---|
| `whisper_model_path` | Путь к модели Whisper |
| `genapi_network_id` | Модель GenAPI: `gpt-4-1` или `gemini-2-5-flash-lite` |
| `use_gpu` | `true` — Whisper на GPU (нужна NVIDIA + CUDA), `false` — CPU |
| `whisper_language` | Язык транскрипции: `ru`, `en`, `auto` |

## 🔑 Получение ключа GenAPI

1. Зарегистрируйтесь на [gen-api.ru](https://gen-api.ru)
2. Перейдите в раздел **API Keys**
3. Создайте ключ и скопируйте его
4. Вставьте в `.env`: `GEN_API_KEY=sk-...`
